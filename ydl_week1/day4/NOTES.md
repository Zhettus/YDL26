### Q1 — Заполнение возраста по группам: почему так, что делают функции?

```python
df['Age'] = df.groupby(['Title', 'Pclass'])['Age'].transform(
    lambda x: x.fillna(x.median()))
```

**Почему именно так, а не одним средним для всех?**  
Age содержит 177 пропущенных значений (~20%). Самое простое — заполнить
глобальным средним (~30 лет). Но это неточно: "Master" — обращение к мальчику
до ~13 лет, а "Mr" — взрослый мужчина. Заполнить "Master" средним в 30 лет —
прямая ошибка. Группируя по Title (косвенно кодирует пол + возрастную группу) и
Pclass (богатые пассажиры 1-го класса были в среднем старше), мы получаем
гораздо более реалистичную оценку возраста для каждого пассажира.

**Как это влияет на модель?**  
Age — важный предиктор: детей и женщин сажали в шлюпки первыми. Неправильный
возраст → модель неверно оценивает вероятность выживания.

**Функции:**

| Функция | Что делает |
|---|---|
| `df.groupby(['Title', 'Pclass'])` | Разбивает датафрейм на группы по уникальным комбинациям Title + Pclass |
| `['Age']` | Выбирает столбец Age внутри каждой группы |
| `.transform(lambda x: ...)` | Применяет функцию к каждой группе, но **возвращает Series с тем же индексом**, что и исходный df — поэтому результат можно напрямую присвоить обратно в df['Age'] |
| `x.fillna(x.median())` | Внутри каждой группы заменяет NaN медианой этой же группы |

### Q2 — Safety fallback: что делает эта строка?

```python
df['Age'] = df['Age'].fillna(df['Age'].median())
```

После группового заполнения некоторые строки могут **всё ещё** содержать NaN.
Это происходит, если в целой группе (например, Title=Rare + Pclass=3) **все**
значения Age были NaN — тогда медиана группы тоже NaN и заполнять нечем. Эта
строка — страховка: заполняем оставшиеся NaN глобальной медианой по всему
датасету. Без неё LinearRegression упадёт с ошибкой на строках с NaN.

### Q3 — Что происходит с Embarked, как это помогает?

```python
df['Embarked'] = df['Embarked'].fillna(df['Embarked'].mode()[0])
df['Embarked'] = df['Embarked'].map({'S': 0, 'C': 1, 'Q': 2})
```

**Строка 1 — заполнение пропусков:**  
В датасете 2 пассажира без порта посадки. `.mode()[0]` возвращает самое частое
значение ('S' — Southampton). Индекс `[0]` нужен потому, что `.mode()` возвращает
Series, а нам нужно скалярное значение.

**Строка 2 — кодирование в числа:**  
LinearRegression работает только с числами, строки 'S', 'C', 'Q' она не понимает.
Мы заменяем их на 0, 1, 2. Это не означает "C в два раза больше S" — это просто
числовые метки. Модель улавливает сигнал: C (Cherbourg) = больше богатых
пассажиров 1-го класса = выше шанс выжить; Q (Queenstown) = в основном 3-й
класс = ниже шанс выжить.

### Q4 — Что такое Fare, как помогает, как используется?

Fare — **цена билета** в фунтах стерлингов.

Почему полезен: цена билета очень сильно коррелирует с классом (Pclass), но даёт
дополнительную детализацию. Внутри 1-го класса пассажир, заплативший £500, имел
каюту ближе к шлюпочной палубе, чем заплативший £50. Fare добавляет нюанс туда,
где Pclass (только 1/2/3) слишком грубый.

Как используется:
- В train.csv пропусков нет.
- В test.csv один пассажир без Fare → `df['Fare'].fillna(df['Fare'].median())`
  заполняет его медианой, чтобы модель не упала при предсказании.
- Дальше Fare входит в список FEATURES как есть — числовой столбец, кодирования
  не нужно.

### Q5 — `.values` — это то же самое, что `axis=1`?

**Нет, это совершенно разные вещи.**

**`.values`** — свойство pandas DataFrame/Series, которое конвертирует таблицу в
«голый» NumPy массив, убирая названия колонок и индексы строк. sklearn ожидает
NumPy arrays, а не pandas DataFrames, поэтому `.values` нужен для совместимости.

```python
X = train_clean[FEATURES].values  # shape (891, 11), тип numpy.ndarray
```

**`axis=1`** — параметр многих pandas-функций, который задаёт **направление
операции**:
- `axis=0` — операция «вдоль строк», то есть по каждому столбцу
- `axis=1` — операция «вдоль столбцов», то есть по каждой строке

```python
df.drop('Name', axis=1)   # удалить столбец (ось столбцов = axis=1)
df.sum(axis=1)            # сумма по строке (сложить все столбцы)
```

Итог: `.values` меняет **тип объекта** (DataFrame → ndarray), а `axis=1` задаёт
**направление** операции внутри pandas. Никакой связи между ними нет.

### Q6 — Что такое cross_val_score? Нарушает ли правила задачи? Что делает эта ячейка?

**cross_val_score** — функция из sklearn для перекрёстной проверки (cross-
validation). Она импортирована, но в итоге не используется в коде (используется
`StratifiedKFold` напрямую внутри функции) — это неиспользуемый импорт.

**Нарушает ли правила?** Нет. Правило — «только LinearRegression». Cross-
validation — это способ **оценить** модель, а не другая модель. Мы по-прежнему
обучаем только LinearRegression.

**Что делает вся ячейка:**

Здесь реализована **5-кратная стратифицированная перекрёстная проверка** (5-fold
stratified cross-validation). Смысл:

```
Обучающие данные (891 строка)
─────────────────────────────
│   Fold 1  │  Train (80%)  │  Val (20%)  │  → accuracy_1
│   Fold 2  │  Train (80%)  │  Val (20%)  │  → accuracy_2
│   Fold 3  │  Train (80%)  │  Val (20%)  │  → accuracy_3
│   Fold 4  │  Train (80%)  │  Val (20%)  │  → accuracy_4
│   Fold 5  │  Train (80%)  │  Val (20%)  │  → accuracy_5
                                    Среднее = честная оценка
```

**Зачем нужно:** Если оценивать точность на тех же данных, на которых учились,
модель «уже видела» ответы — получится завышенный результат. CV каждый раз
проверяет модель на данных, которые она **не видела при обучении** — это честная
оценка того, как модель будет работать на kaggle-тесте.

**Слово "Stratified"** означает, что в каждом fold соотношение выживших / не
выживших сохраняется примерно таким же, как в полном датасете (~38% выжили).

### Q7 — Что такое threshold? Что такое accuracy? В чём разница?

**Accuracy (точность)** — доля правильных предсказаний от всех предсказаний.

```
Accuracy = (верно предсказанные) / (все пассажиры)
         = (TP + TN) / (TP + TN + FP + FN)
```

Например, accuracy = 0.82 означает: из 891 пассажира модель правильно предсказала
исход для ~730.

**Threshold (порог)** — значение, выше которого мы считаем предсказание = 1
(выжил).

LinearRegression выдаёт **непрерывное число**, например 0.23, 0.67, 0.81. Это не
вероятность, просто число. Чтобы получить ответ 0 или 1, нужно решить: "при каком
значении считать пассажира выжившим?"

```
raw_pred = 0.67
threshold = 0.5  →  0.67 >= 0.5  →  prediction = 1 (выжил)
threshold = 0.7  →  0.67 < 0.7   →  prediction = 0 (не выжил)
```

**В чём разница между ними:**

| | Threshold | Accuracy |
|---|---|---|
| Что это | Правило перевода числа → 0/1 | Метрика качества модели |
| На что влияет | На то, кого мы называем выжившим | На то, насколько мы правы |
| Как задаётся | Мы выбираем сами | Считается по результатам |

Взаимосвязь: изменяя threshold, мы меняем predictions → меняется accuracy.
Именно поэтому мы перебираем пороги от 0.30 до 0.70 и ищем тот, при котором
accuracy максимальна.
